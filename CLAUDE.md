# Claude Code notes — Location Scraper

INDATEL Labs internal tool. Two variants share one codebase:

| Variant | Purpose | Entry | Code |
|--------|---------|-------|------|
| Legacy | Auto-discovery + try-all-strategies | `app.py` | `scraper/` |
| AI    | LLM-orchestrated (Gemini Flash) | `ai_version/app_ai.py` | `ai_version/scraper_ai/` + `scraper/` |

The AI version is the one being actively developed and shipped to employees. Legacy is kept working as a fallback.

## Repository

- **GitHub**: https://github.com/corbandavidson/indatel-location-scraper (private)
- **gh CLI** is installed and authed as `corbandavidson`. Use `gh` for any GitHub work.
- **Releases** are how employees get updates. The installer attached to each release is what they download.

## Editing code — the only rule that matters

**Edit source files** (`ai_version/`, `scraper/`, `config/`). **Never** edit files inside `dist/LocationScraperAI/` — that's a built artifact, regenerated by the build pipeline. The build script syncs source → dist for you.

`dist/` and `build/` are gitignored (~883MB raw). They're regenerated as needed.

## Shipping an update — the workflow

After making code changes, one command publishes them to every employee:

```powershell
.\build.ps1 -Version 1.0.X -Publish
```

That command:
1. Bumps `ai_version/version.py` to the new version
2. Syncs `ai_version/`, `scraper/`, `config/` into `dist/LocationScraperAI/app/` via `robocopy /MIR`
3. Clears every `__pycache__/` directory in source and dist
4. Compiles the Inno Setup installer (ISCC.exe) → `installer/Output/LocationScraperAI-Setup-X.Y.Z.exe`
5. `git add` + commit the version bump, create `vX.Y.Z` tag, push main + tag
6. `gh release create vX.Y.Z` with the installer as the attached asset

Employees see "Update available" in the app's sidebar on next launch — the app polls `api.github.com/repos/corbandavidson/indatel-location-scraper/releases/latest` (cached for 1 hour via `@st.cache_data`).

**Local-build-only** (test before publishing): drop `-Publish`. Run with `-SkipBump` if rebuilding the same version after fixing the Inno script.

## When build.ps1 is NOT enough

The script only handles source-only changes (`.py` edits). Run the slower full PyInstaller rebuild first if any of these change:
- `requirements.txt` (new dependency)
- `desktop_ai.py` (the launcher exe)
- `LocationScraperAI.spec` (PyInstaller spec)
- Playwright browsers / embedded Python version

Full rebuild path: run `build.bat` (legacy) or the AI spec via `pyinstaller ai_version/LocationScraperAI.spec`. Then run `build.ps1 -Version ... -Publish`.

## Architecture pointers

- `scraper_ai/planner.py` — Gemini wrapper. Has `find_locator_url`, `pick_strategy`, `extract_stores_from_html`, `estimate_store_count`, `find_better_locator_url`. Auto-falls-back across Gemini model IDs on 404.
- `scraper_ai/orchestrator.py` — Glues the planner around the legacy scraper. After cleaning, runs `_validate_locations()` and retries with an alternate URL if results look geo-filtered (< 20% of expected count, or national chain with < 10 states).
- `scraper_ai/stealth.py` — Playwright with anti-bot patches (10-point init script + `--disable-http2`). Auto-triggered when initial render returns a blocked/empty page. Defeats Cloudflare challenge tier but NOT Costco-level network fingerprinting.
- `scraper/` (the legacy package) is shared — **don't fork it**. AI version reaches into it via `sys.path` adjustment.

## Settings

User-facing API key + output prefs live at `%USERPROFILE%\.location_scraper\settings.json`. The installer leaves this alone on uninstall.

## Branding

Customer-facing language never mentions Gemini, the model name, or AI vendor specifics. UI says "AI", help text says "Contact INDATEL Labs for an API key". Keep it that way.

## Gotchas that will bite you

- **Streamlit hot-reload doesn't reload imported modules.** When the user has the app running and you edit `scraper_ai/*.py`, they MUST relaunch the exe. Hot-reload only catches changes to the top-level `app_ai.py`.
- **PowerShell `ErrorActionPreference=Stop` treats git's stderr text as fatal.** `git push` writes "Everything up-to-date" to stderr → script abort. `build.ps1` has an `Invoke-Native` helper that merges stderr and gates on `$LASTEXITCODE`. Use the same pattern for any new git/gh calls.
- **Scripts are blocked by default.** First time on a new machine: `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`.
- **Inno Setup path is hardcoded** to `C:\Program Files (x86)\Inno Setup 6\ISCC.exe`. If the user installs to a different path, update `$IscPath` in `build.ps1`.
- **`__pycache__/` staleness** is the #1 cause of "I changed the code but nothing happened" reports. The build script clears it everywhere; if running source by hand, do the same.

## Quick reference

```powershell
# Make a change, test locally
.\build.ps1 -Version 1.0.1            # builds installer at installer\Output\

# Happy with it — publish
.\build.ps1 -Version 1.0.1 -Publish   # commits, tags, pushes, creates release

# Re-build the same version without bumping (e.g. fixed installer config)
.\build.ps1 -Version 1.0.1 -SkipBump
```
